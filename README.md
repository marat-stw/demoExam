## Модули 2-3 — База данных и SQL-запрос

В папке лежат 2 SQL-файла — это разные варианты одного и того же решения, отличаются способом импорта данных из `Заказчики.json`:

- **module2-3 (быстрый импорт данных из JSON)** — импорт данных выполняется напрямую через функцию `OPENJSON()`, без предварительного создания строк `INSERT`. Быстрее, но требует, чтобы JSON-файл был доступен серверу по указанному пути.
- **module2-3 (обычный инсерт).sql** — классический способ: данные из JSON преобразованы в обычные `INSERT INTO ...` команды. Чуть длиннее по коду, но не зависит от пути к файлу — можно просто выполнить скрипт на любом сервере.

**Как использовать:**
1. Открыть SQL Server Management Studio (SSMS)
2. Подключиться к серверу
3. Открыть один из `.sql`-файлов
4. Выполнить весь скрипт (`Execute` / F5) либо выполнять по частям выделяя нужную часть 

> Если используете вариант «через JSON» — убедитесь, что путь к файлу `Заказчики.json` в скрипте указан верно для вашей машины.

## Модуль 4 — Приложение

Папка **`Кофейня/`** — проект на C# (WinForms).

**Как открыть и запустить:**
1. Открыть файл `Кофейня.sln` в Visual Studio
2. Перед запуском — открыть файл `Data/DatabaseHelper.cs` и поменять строку подключения:
   ```csharp
   private static readonly string ConnectionString =
       "Server=localhost;Database=Производство;Integrated Security=True;";
   ```
   Заменить `localhost` на имя SQL Server, который используется на экзаменационном месте.
3. Убедиться, что в БД уже выполнен соответствующий `module2/module2-3` скрипт (таблицы и данные созданы)
4. Запустить проект (F5)

**Функционал приложения:**
- Форма авторизации с проверкой логина/пароля
- Капча-пазл (нужно собрать картинку из 4 фрагментов перетаскиванием) перед входом
- Блокировка учётной записи после 3 неудачных попыток (по паролю или по капче)
- Панель администратора: просмотр, добавление, редактирование пользователей, снятие блокировки
